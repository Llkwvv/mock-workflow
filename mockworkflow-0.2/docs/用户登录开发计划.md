## 1. 项目概述

Mockworkflow 是一个面向数据科学家的 Mock 数据生成平台，提供 Web 控制台界面，支持样本管理、任务调度、数据生成引擎、插件扩展等功能。

用户登录模块是平台的基础安全设施，负责：
- 控制台的访问认证
- API 接口的权限校验
- 用户会话的生命周期管理

## 2. 功能需求

### 2.1 核心功能

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F-01 | 密码登录 | P0 | 用户通过密码登录 Web 控制台 |
| F-02 | 会话管理 | P0 | 基于 Token 的会话机制，支持 7 天过期 |
| F-03 | API 认证 | P0 | 所有 API 请求需携带有效凭证 |
| F-04 | 自动会话续期 | P0 | 通过 Header 认证时自动创建 Cookie 会话 |
| F-05 | 登录状态检查 | P1 | 前端定时检查认证状态，失效时跳转登录 |
| F-06 | 退出登录 | P1 | 清除服务端和客户端会话 |

### 2.2 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF-01 | 安全 | 密码使用 HMAC 常量时间比较，防止时序攻击 |
| NF-02 | 安全 | Session Token 使用 `secrets.token_urlsafe(32)` 生成 |
| NF-03 | 安全 | Cookie 设置 HttpOnly，防止 XSS 窃取 |
| NF-04 | 安全 | 生产环境需启用 HTTPS + Secure Cookie |
| NF-05 | 性能 | 认证中间件开销 < 1ms（内存会话校验） |
| NF-06 | 兼容 | 支持 API Header 认证和浏览器 Cookie 认证两种方式 |

## 3. 现有实现评估

### 3.1 后端认证模块 (`backend/app/auth.py`)

**已完成：**
- ✅ `SessionStore` 类：内存 + JSON 持久化会话存储
- ✅ `verify_password()`：HMAC 常量时间比较
- ✅ `create_session()` / `validate_session()` / `revoke_session()` 完整生命周期
- ✅ `AuthMiddleware`：FastAPI 中间件，支持路径白名单
- ✅ 支持 `X-Password` 和 `Authorization: Bearer` 两种 Header 认证

**待优化：**

- ⚠️ 会话存储使用单文件 JSON，高并发下可能存在竞争
- ⚠️ 无登录失败次数限制（暴力破解防护）
- ⚠️ 无登录审计日志

### 3.2 认证路由 (`backend/app/routers/auth.py`)

**已完成：**
- ✅ `POST /api/auth/login` — 密码登录
- ✅ `POST /api/auth/logout` — 注销
- ✅ `GET /api/auth/me` — 检查认证状态

**待补充：**
- ⚠️ 无登录失败锁定机制
- ⚠️ 无登录成功/失败事件通知

### 3.3 前端登录界面 (`frontend/index.html` + `main.js`)

**已完成：**
- ✅ 登录弹窗 UI（密码输入 + 登录按钮 + 错误提示）
- ✅ `doLogin()` — 登录请求 + Cookie 存储
- ✅ `checkAuth()` — 页面加载时检查认证
- ✅ 401 自动重试队列（请求拦截后自动重放）
- ✅ 全局 API 封装自动处理认证

**待优化：**
- ⚠️ 无记住密码功能
- ⚠️ 无登录页面美化（当前为极简弹窗）
- ⚠️ 登录过期后用户体验可优化（Toast 提示更友好）

## 4. 技术方案

### 4.1 认证流程

```
┌──────────┐         ┌──────────┐         ┌──────────┐
│  Browser  │         │   API     │         │  Session  │
│  (Front)  │ ──────▶ │  Gateway  │ ──────▶ │  Store    │
└──────────┘         └──────────┘         └──────────┘
     │                       │                     │
     │  1. POST /login       │                     │
     │  {password}           │                     │
     │◀──────────────────────│                     │
     │  2. Set-Cookie: token │                     │
     │                       │                     │
     │  3. 后续请求携带 Cookie                     │
     │──────────────────────▶│  4. 验证 Token      │
     │                       │────────────────────▶│
     │                       │◀────────────────────│
     │                       │  5. 返回结果         │
     │◀──────────────────────│                     │
```

### 4.2 双认证模式

| 场景 | 认证方式 | 适用场景 |
|------|----------|----------|
| 浏览器访问 | Cookie (mockworkflow_session) | Web 控制台 |
| API 调用 | X-Password / Authorization: Bearer | 自动化脚本、第三方集成 |

### 4.3 密码存储

- 当前方案：配置文件中明文密码（`config.py` → `web_password`）
- 后续建议：bcrypt/argon2 哈希存储

## 6. API 设计规范

### 6.1 认证相关 API

#### `POST /api/auth/login`
```json
// Request
{ "password": "string (required)" }

// Response 200
{ "success": true, "message": "Login successful." }
// Set-Cookie: mockworkflow_session=<token>; HttpOnly; SameSite=Lax

// Response 401
{ "success": false, "message": "Invalid password." }
```

#### `POST /api/auth/logout`
```json
// Response 200
{ "success": true, "message": "Logout successful." }
// Delete-Cookie: mockworkflow_session
```

#### `GET /api/auth/me`
```json
// Response 200
{ "authenticated": true }
```

### 6.2 Header 认证方式

```
X-Password: <明文密码>
# 或
Authorization: Bearer <明文密码>
```

> ⚠️ Header 认证仅用于 API 自动化场景，不推荐浏览器端使用。

## 7. 安全考虑

### 7.1 当前已实现

- ✅ HMAC 常量时间密码比较（防时序攻击） → 升级为 **bcrypt** 哈希
- ✅ Session Token 使用密码学安全随机数 (`secrets.token_urlsafe(32)`)
- ✅ HttpOnly Cookie（防 XSS 窃取）
- ✅ SameSite=Lax（防 CSRF）
- ✅ 路径白名单豁免（登录页、健康检查等）
- ✅ IP登录失败锁定（5次失败，15分钟锁定）
- ✅ 登录审计日志（JSON行格式，记录成功/失败/注销）
- ✅ CSRF Token 保护（登录API）
- ✅ 安全响应头（HSTS、CSP、X-Frame-Options、X-Content-Type-Options等）
- ✅ 生产环境 Secure Cookie（环境变量控制）

### 部署说明

1. 设置环境变量 `MOCKWORKFLOW_WEB_PASSWORD` 为登录密码
2. （可选）设置 `MOCKWORKFLOW_SECURE_COOKIES=1` 启用 Secure Cookie
3. 启动服务：`.venv/bin/python3 start_backend.py`
4. 访问前端：http://localhost:8000
