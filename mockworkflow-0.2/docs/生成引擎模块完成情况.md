### 策略管道完整流程

```
第0层  全空列 → 复制空值
第1层  自增ID / identity语义 → row index
第1b层 中文证件号 → cn_identifiers (含真实校验)
第2层  日期时间/birthdate → 样本日期范围内随机
第3层  布尔/flag → 0/1 随机
第4层  数值/坐标 → 分布采样 或 观测边界内随机
第5层  高价值Faker语义 → 车牌/电话/邮箱/URL/车辆型号/方向
第6层  实体伪造 → 人名/公司/机构/地址 (绝不复用真实值)
第7层  结构化标识符 → 模板+随机数字替换
第8层  枚举值 (唯一复用路径) → 频率加权采样真实值
第9层  非枚举自由文本 → 马尔可夫伪造 (绝不复用)
第10层 列名启发式 → Faker按列名语义
第11层 兜底 → 合成短句
```

```
任务完成 ──→ self_learn.learn_from_task()
              ├── RuleStore.upsert_fields() → JSON规则库
              ├── RuleIndexer.index_rules()  → Chroma "rules" 集合
              └── SampleIndexer.index_profile() → Chroma "samples" 集合

新任务 ───→ resolve_fields()
            ├── RuleStore.find()         ← JSON精确/别名匹配
            ├── RuleIndexer.search()     ← 语义向量检索
            ├── SampleIndexer.search()   ← 相似列名/样本检索
```

## 总体评估

### 完成度：~95%

### 架构亮点

1. **严格模式**：11层策略级联中仅第8层（枚举）可复用真实样本值，其余层全部伪造，从设计上杜绝数据泄露
2. **跨字段一致性**：身份证→出生/性别/年龄/籍贯、省市区层级、邮编/信用代码跟随区域，全部使用真实行政区划数据
3. **真实校验算法**：身份证(GB 11643)、统一社会信用代码(GB 32100)、银行卡(Luhn)均通过官方校验
4. **质量反馈闭环**：生成→验证→高分规则持久化→下次直接命中，系统随使用持续进化
5. **多层解析管线**：规则库缓存 → RAG语义检索 → LLM解析 → 规则引擎兜底，逐层降级保证可用性
6. **模型轮转**：完整的failure tracking、blacklist/degraded/skip_auto分级管理、分组选择、持久化上次成功模型
7. **自我学习**：每次成功任务后自动索引规则和样本profile到向量库
